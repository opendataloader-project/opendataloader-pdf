## Key Decisions Made

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| In-memory job store (no SQLite) | Zero deps, simple v0 | SQLite persistence |
| Content-hash dedup on DONE jobs only | Failed jobs should be retryable | Dedup all terminal states |
| Per-job `_status_lock` (not global) | Fine-grained, avoids convoy | Single global lock |
| `daemon=True` threads | Clean process shutdown, no state to flush | Non-daemon (would hang shutdown) |
| `get_job_resource` returns error strings (not raises) | MCP resource protocol requires valid response | Raising from resource handler |
| LLM fallback stub (Task 7, Plan A) | Real LLM integration deferred to Plan C | Wire real LLM now |
